# Security Policy

## Supported Versions

Vvveb CMS currently supports the following versions with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

**Only the latest version (currently v1.0.8 released February 19, 2026) and future 1.0.x releases receive regular security patches.**  
Older versions in the 1.0 series may receive critical fixes on a case-by-case basis.

## Reporting a Vulnerability

We take the security of Vvveb CMS very seriously. If you discover a security vulnerability, we appreciate your responsible disclosure.

**Please do not report vulnerabilities through public GitHub Issues.**

The best way to report:
1. Go to the **[Security](https://github.com/givanz/Vvveb/security)** tab in this repository.
2. Click **"Report a vulnerability"**.

   → This creates a **private Security Advisory** (visible only to maintainers) so we can discuss it confidentially.

Alternative (if you prefer):
- Contact the maintainer directly via GitHub: **[@givanz](https://github.com/givanz)** (send a private message or open an issue with prefix `[SECURITY]`).

### Our response timeline
- **Acknowledgment**: within 48 hours  
- **Fix & confirmation**: up to 30 days (faster for critical vulnerabilities)  
- **Credit**: We will publicly credit you (unless you request to remain anonymous)

Thank you for helping keep Vvveb secure! ❤️
